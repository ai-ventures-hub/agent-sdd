# Product Overview

## Purpose
Defines the project’s mission, target users, and goals to guide `/sdd-task --roadmap` and `/sdd-task --spec` workflows. Stored in `.claude/product/overview.md` for reference by `.claude/agents/context-fetcher.md`.

## Usage
- Replace placeholders with project-specific details.
- Keep mission statement concise (one sentence).
- List 2-4 primary user types.
- Set 3-5 specific, measurable goals.
- Updated by `.claude/agents/file-creator.md` during `/sdd-task --roadmap`.

## Mission
[Write a one-sentence description of what your product does and why it matters]

**Examples:**
- Help small businesses manage inventory efficiently through automated tracking.
- Enable developers to deploy applications faster with streamlined CI/CD workflows.
- Connect pet owners with trusted veterinarians via a mobile platform.

## Target Users
- [Primary user type: Who uses this most often?]
- [Secondary user type: Who else benefits directly?]
- [Tertiary user type: Other key stakeholders?]

**Examples:**
- **Web App**: End users, admin users, content managers.
- **Developer Tool**: Frontend developers, DevOps engineers, team leads.
- **Mobile App**: Users (age/demographic), business owners, support staff.
- **API/Service**: Integration developers, product teams, system administrators.

## Goals
1. [Specific, measurable UX goal]
2. [Specific, measurable functionality/performance goal]
3. [Specific, measurable adoption/business goal]

**Examples:**
- Reduce setup time from 2 hours to 15 minutes by Q3 2025.
- Achieve 95% uptime over 6 months.
- Enable users to complete tasks 50% faster.
