# /sdd-review-code <file-or-dir-path> [--fix] [--report-only]
Review code for **Theme Standards** compliance and (optionally) auto‑fix issues.

---

## Purpose
Ensure UI code follows `.agent-sdd/standards/theme-standards.md` and the active theme at  
`.agent-sdd/standards/theme-files/[theme-name]/theme.css` (source of truth).

---

## What this checks
1. **Theme source** (in priority order):
   - **1)** `.agent-sdd/standards/theme-files/[theme-name]/theme.css` ← generated by `/sdd-apply-theme`
   - 2) `.agent-sdd/standards/theme-files/[theme-name]/syntax-custom.css` (legacy fallback)
   - 3) `.agent-sdd/standards/theme-files/[theme-name]/app.css` (legacy fallback)
2. **Allowed utilities** from `theme-standards.md` (color/typography allow‑lists).
3. **Disallowed styles**:
   - Raw hex/rgb colors in JSX/CSS (except inside theme files).
   - Tailwind color classes not on the allow‑list (e.g., `bg-sky-500`).
   - Missing dark mode pairing when applicable (`dark:` variants for text/bg).
   - Missing focus styles on interactive elements.
4. **Layout/spacing**:
   - 4px grid (e.g., `p-1/2/3`, `gap-1/2/3`, etc.).
   - Mobile‑first usage (default classes, then `sm:`, `md:`, `lg:`).
5. **A11y**:
   - WCAG 2.1 AA hints (focus-visible, touch targets ≥ 40px).
   - ARIA roles/labels where required components exist.

---

## Behavior
- **Default**: analyze and suggest fixes (no writes).
- `--fix`: make `.bak` backup, then apply safe text edits (utility replacements only).
- `--report-only`: print report only (same as default, explicit).

---

## Steps (what the agent does)
1. **Determine active theme name** from `.agent-sdd/standards/theme-standards.md` title  
   (e.g., `# Theme Standards (minimal)` ⇒ `theme-name = minimal`).
2. **Load theme tokens**:
   - Prefer `theme-files/[theme-name]/theme.css`; parse CSS variables:
     - `--color-primary`, `--color-secondary`, `--color-success`, `--color-error`,
     - `--text-default`, `--text-on-primary`, `--bg-default`.
   - Build utility mapping:
     - `bg-primary` ⇒ `var(--color-primary)`
     - `bg-secondary` ⇒ `var(--color-secondary)`
     - `bg-success` ⇒ `var(--color-success)`
     - `bg-error` ⇒ `var(--color-error)`
     - `text-default` ⇒ `var(--text-default)`
     - `text-on-primary` ⇒ `var(--text-on-primary)`
3. **Read allow‑lists** from `theme-standards.md` (colors & typography).
4. **Scan target** `<file-or-dir-path>`:
   - File types: `*.tsx`, `*.ts`, `*.jsx`, `*.js`, `*.css`, `*.scss`.
   - Detect:
     - Disallowed Tailwind colors (not on allow‑list).
     - Inline styles with color literals (hex/rgb/oklch).
     - Missing `dark:` variant where text/background is themed.
     - Missing focus styles on buttons/links/inputs.
     - Spacing not aligned to 4px grid.
5. **Propose or apply**
