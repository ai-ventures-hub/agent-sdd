## Purpose
Ensure UI code follows `.agent-sdd/standards/theme-standards.md` and the active theme at  
`.agent-sdd/standards/theme-files/[theme-name]/theme.css` (**source of truth**).

---

## What this checks
1. **Theme source priority** (use first file that exists):
   - **1)** `.agent-sdd/standards/theme-files/[theme-name]/theme.css` ŌåÉ generated by `/sdd-apply-theme`
   - 2) `.agent-sdd/standards/theme-files/[theme-name]/syntax-custom.css` (legacy fallback)
   - 3) `.agent-sdd/standards/theme-files/[theme-name]/app.css` (legacy fallback)
2. **Allowed utilities** from `theme-standards.md` (color/typography allowŌĆælists).
3. **Disallowed styles**:
   - Raw hex/rgb/oklch colors in JSX/CSS (except inside theme files).
   - Tailwind color classes not on the allowŌĆælist (e.g., `bg-sky-500`).
   - Missing dark mode pairing when applicable (`dark:` variants for text/bg).
   - Missing focus styles on interactive elements.
4. **Layout/spacing**:
   - 4px grid (e.g., `p-1/2/3`, `gap-1/2/3`, etc.).
   - MobileŌĆæfirst usage (default classes, then `sm:`, `md:`, `lg:` overrides).
5. **A11y**:
   - WCAG 2.1 AA hints (focus-visible, touch targets Ōēź 40px).
   - ARIA roles/labels where required.

---

## Behavior
- **Default**: analyze and suggest fixes (no writes).
- `--fix`: make `.bak` backup, then apply safe text edits (utility replacements only).
- `--report-only`: print report only (same as default, explicit).

---

## Steps (what the agent does)
1. **Determine active theme** from `.agent-sdd/standards/theme-standards.md` title  
   (e.g., `# Theme Standards (minimal)` ŌćÆ `theme-name = minimal`). If no name is present, assume `minimal`.
2. **Load theme tokens** from the first existing file (priority list above):
   - Parse CSS variables:
     - `--color-primary`, `--color-secondary`, `--color-success`, `--color-error`,
     - `--text-default`, `--text-on-primary`, `--bg-default`.
   - Build utility mapping:
     - `bg-primary` ŌćÆ `var(--color-primary)`
     - `bg-secondary` ŌćÆ `var(--color-secondary)`
     - `bg-success` ŌćÆ `var(--color-success)`
     - `bg-error` ŌćÆ `var(--color-error)`
     - `text-default` ŌćÆ `var(--text-default)`
     - `text-on-primary` ŌćÆ `var(--text-on-primary)`
3. **Read allowŌĆælists** from `theme-standards.md` (colors & typography).
4. **Scan target** `<file-or-dir-path>`:
   - File types: `*.tsx`, `*.ts`, `*.jsx`, `*.js`, `*.css`, `*.scss`.
   - Detect:
     - Disallowed Tailwind colors (not on allowŌĆælist).
     - Inline styles with color literals (hex/rgb/oklch).
     - Missing `dark:` variant where themed text/bg is used.
     - Missing focus styles on buttons/links/inputs.
     - Spacing not aligned to 4px grid.
5. **Propose or apply fixes**:
   - Replace **raw colors** with approved utilities (e.g., `bg-[#2563eb]` ŌćÆ `bg-primary`).
   - Replace **disallowed utilities** with allowed equivalents by semantic (e.g., `bg-blue-500` ŌćÆ `bg-primary`).
   - Add `dark:` companion classes if the element is themed text/bg and a dark token exists.
   - Add `focus-visible` outline utilities per project standard.
6. **Write changes if `--fix`**:
   - Create `.bak` once per file.
   - Write updated content.
   - Suggest commit: `style: Update <path> for theme compliance`.
7. **Output report**:
   - Issues found (before/after).
   - Fixes applied (if `--fix`).
   - Reminder to import the theme once if missing:
     ```css
     @import "../../.agent-sdd/standards/theme-files/[theme-name]/theme.css";
     ```

## Constraints
- **Never modify logic** or component semantics.
- **Only** adjust classes/styles and add minimal a11y utilities.
- **Back up** as `.bak` before writing.
- **Do not** edit files under `.agent-sdd/standards/theme-files/`.
